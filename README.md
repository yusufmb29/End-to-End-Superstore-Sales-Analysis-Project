# Superstore-Sales-EDA-Project
Superstore Sales raw dataset cleaning and Exploratory Data Analysis using PostgreSQL .

## 👉 Project Overview / Objective
This project focuses on performing end-to-end data analysis on a retail Superstore dataset using SQL.
The objective is to clean the data, perform exploratory data analysis (EDA), create reusable analytical views, and derive business insights related to sales, profitability, customers, products, and regional performance.

## ☺️ Dataset Description
The dataset contains historical sales data of a Superstore, including information about orders, customers, products, categories, discounts, profit, and geographic location (state-level).

## 🛠️ Tools & Technologies Used
1. SQL (PostgreSQL)
2. pgAdmin
3. Github

## ➡️ Project Workflow
The project was completed in the following steps:
1. Data loading and table creation
2. Data cleaning (handling nulls, data types, formatting, standardizing)
3. Exploratory Data Analysis (EDA)
4. Creation of reusable SQL views for reporting
5. Business-focused analysis and insights

## 👍 Key SQL Concepts Used
Aggregations (SUM, AVG, COUNT)
GROUP BY
CASE WHEN
Window functions (RANK, LAG, NTILE)
CTEs (WITH clause)
Views
Date functions (EXTRACT, DATE_TRUNC)

## 🪟 Views Created
1. monthly_sales_profit_trend
2. state_performance_summary

## 💡Key Business Insights derived
1. Technology category generates the highest sales, while Furniture has lower profit margins.
2. California State has most orders as well as high sales and profit.
3. Sales show a clear upward trend over the years.
4. Frequency of Frequent Customers is high as compared to Ocassional and One-Time Customers.
